# Security Policy

## Scope

This repository is a workflow kit, not a hosted service, but security issues still
matter. Report vulnerabilities in scripts, unsafe workflow guidance, or anything
that could reasonably cause users to leak secrets, damage repositories, or run
unsafe commands.

## How to Report

- Do not post exploit details in a public issue.
- If a private contact channel is listed in repository metadata or the maintainer
  profile, use that channel.
- If no private channel is available, open a GitHub issue requesting a security
  contact path and include no sensitive details.

## What to Include

- A short description of the issue and affected files or commands.
- Impact and likely preconditions.
- Reproduction steps or a minimal proof of concept, shared privately when possible.
- Suggested mitigations, if you have them.

## Response Expectations

This is a solo-maintained project, so response times are best-effort. The
maintainer will triage good-faith reports, confirm severity when possible, and
fix or document the issue as capacity allows.
